В реакторе принята модульная структура. Основное приложение также является модулем, как и другие составные части, предназначенные для выполнения определенного набора функций.
Каждый модуль задается конфигурацией в виде файла в формате JSON в каталоге "config". Файлы модулей самого реактора находятся в каталоге Reactor, приложения в каталоге Mod.
Создание нового модуля и добавление в приложение на примере тестового модуля состоит из двух частей - создание файла конфигурации - test.json и каталога Mod\Test, внутри которого должны быть каталог - tpl - для шаблонов и файл Module.php с классом, унаследованным от \Reactor\Application\Module
Файл конфигурации задает название модуля для дальнейшего его использования и путь до основного класса модуля -
"modules": {
    "test": "\\Mod\\Test\\Module"
}
кроме блока с зарезервированным названием "modules" в файле конфигурации можно использовать также блок под названием "parameters" для задания параметров у объекта приложения, например, $app->view и блок "services" с описанием сервисов - составных частей отдельного модуля
