Приложение, созданное при помощи реактора представляет собой набор или коллекцию сервисов. Роль сервиса выполняет инстанс или экземпляр класса с заданным небходимым функционалом для решения определенной задачи. Сервисы для выполнения своей задачи могут также использовать другие сервисы, поэтому приложение начинает работу с запуска необходимого сервиса для выполнения задачи. Для описания связей между сервисами в реакторе существует иерархическая структура, сервисом может быть сервис-контейнер, содержащий собственную коллекцию сервисов. Для определенности, вложенные сервис-контейнеры называются модулями, а сервис-контейнер верхнего уровня - это приложение. 
При запуске создание всех существующих сервисов в составе приложения не является целесообразным, так как это может занять много ресурсов и не все сервисы могут быть нужны на каждом этапе работы приложения. Поэтому при обращении к ним используются прозрачные сервис-провайдеры, которые создают сервисы при необходимости, позволяя их считать уже существующими на момент запуска и инициализации приложения. Сервис-провайдер может использоваться как стандартный, со сценарием и кэшированием, так и самостоятельно написанный, реализующий специальный интерфейс для обеспечения совместимости. 
Для упрощения загрузки сценариев посредством стандартного сервис-провайдера используются конфигурационные файлы в формате JSON.
